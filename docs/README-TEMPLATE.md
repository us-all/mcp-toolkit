## Architecture

Built on `@us-all/mcp-toolkit` (shared lib for the 7-server suite):
- `extractFields` — token-efficient default projections on fat reads
- `aggregate()` — fan-out helper with caveats array for partial-failure surfaces
- `wrapToolHandler` — domain-specific redaction + error extraction
- `search-tools` meta-tool — tool discovery for clients past 100-tool limit

See [mcp-toolkit/STANDARD.md](https://github.com/us-all/mcp-toolkit/blob/main/STANDARD.md).

## Per-repo customization (where to start)

Each repo's "What it does that others don't" should lead with the killer story from the Wave 4 / aggregation tools and Wave 2 prompts:

| Repo | Killer hook |
|---|---|
| datadog-mcp | "Why is this incident happening?" — `analyze-monitor-state`, `slo-compliance-snapshot`, full SLO CRUD (159 tools, no other DD MCP has SLO write or fleet schedules) |
| openmetadata-mcp | OpenMetadata 1.12 Data Contracts + lineage impact + 156 tools across the broadest entity coverage (Data Contracts/Metrics/Search Index/API Endpoint not in official MCP) |
| mlflow-mcp | MLflow 3 trace attachments + summarize-experiment with metric-stats from already-fetched data (zero extra calls) — only third-party MLflow MCP shipping prompt-optimization-jobs + webhooks |
| google-drive-mcp | Power-editing Workspace MCP — formula-aware Sheets, full Slides editing, shared-drive audits the Claude.ai connector deliberately doesn't ship |
| unifi-mcp | Multi-site MSP analytics on the official Site Manager API + Cloud Connector — `site-health-timeline`, `firmware-inventory`, `cross-site-anomaly-detection` (no other UniFi MCP exposes Site Manager analytics) |
| android-mcp | Android diagnostic & forensic MCP — `crash-investigation`, `memory-leak-detection`, `permission-audit` prompts + `device-health` aggregation. Use alongside mobile-mcp; we tell you why it broke. |
| mcp-toolkit | (different — it's a library, not a server) — README is for MCP authors; market the 4 patterns and the cascade workflow |
